* 需求：具体可量化的目标

### 项目痛点

每次重复模块封装：

* 埋点
* `http`请求(`axios`封装)
* 工具方法
* 业务组件库

`git`操作不规范

发布上线耗时：制定标准的上线流程和规范并集成到脚手架

需求：

* 项目/组件的创建能力
* 项目/组件的发布能力

### git 规范

1. git repository
2. fork git repository to your own repository
3. handle your local git repository(风险点)
